# CGV 예매 알림 봇 만들기 (1)

**시리즈 구성**

- **1편 (현재)**: 로컬 환경에서 자동화 구현

- 2편 : 서버 배포 및 24시간 운영

---

아바타: 불과 재가 개봉한지 2주가 되어가는데 아직도 예매를 못했습니다.

아내는 용아맥이 아니면 죽음을 이라고 외치고 있어서 근처 영화관에서 볼수는 없을거같고

이대로는 안되겠다 싶어서 5분마다 CGV를 확인해서 예매가 열렸나 확인하는 자동화를 도전했습니다.

지금 아는 키워드로는 헤드리스 브라우저로 자동화를 하고 크론잡을 통해 배치를 돌리면 되겠다 정도로만 생각했고

부족한 부분은 ai로 채우자라는 마음가짐으로 시작했습니다.

## 1. 무슨 라이브러리를 사용해볼까

```sh
mkdir cgv-checker
cd cgv-checker
npm init -y
npm install playwright node-cron
```

우선 맥북을 켜서 프로젝트를 만들고 playwright와 cron을 설치했습니다.

cron은 설치해두면 나중에 node로 실행할때 5분마다 실행하기라는 옵션을 추가할수 있다 정도로 간단하게 쓰면 되겠지 싶었고

playwright를 선택한 이유는 이전에 써봤던 puppeteer나, 셀레니움, 사이프레스 등 중에서

puppeteer는 크로미움 위주였고 (혹시나 크롬에서 안되면 사파리로도 돌려야하나? 라는 생각때문에)

사이프레스는 테스트를 위한 용도가 더 컸었던거같고 셀레니움은 생소했는데 러닝커브가 높다고 해서 제외했습니다.

그리고 페이지를 보니 지역, 영화관, 상영종류 등을 클릭하고 기다리는데 playwright에서 제공하는 auto wait 기능이

뭔지는 몰랐었지만 클릭하고 화면이 뜰때까지 기다리는걸 sleep 1초씩 늘려가며 하지 않고 알아서 해줄거같아서 선택했습니다.

## 2. 나 대신 알아서 브라우저를 켜고 영화가 열렸는지 확인하기

CGV 예매 페이지가 최근에 개편을 한건지 모바일 친화적이더라구요

이건 크게 문제가 안되는데 혹시 url이 단일이면 어떡하지 걱정을 좀 했었습니다.

다행히 SPA이긴 하지만 라우팅 잘 되어있고 

홈에서부터 찾아갈필요없이 예매페이지가 따로 있긴헀습니다. (https://cgv.co.kr/cnm/movieBook/movie)

물론 페이지에 직접접근하면 종종 예매해야하는 영화 리스트가 안뜨는 경우도 있긴하더라구요

이게 광고차단 때문인건지 플레이라이트로 테스트 했을때 너무 자주 접근해서 봇으로 알고 차단을 하는건지

아니면 그냥 에러인건진 모르겠지만

어차피 5분에 한번씩만 찔러보자 라는 마음으로 시작한거라 이제 이 페이지 위에서 코드를 작성했습니다.

### 1단계 - 브라우저 열기
```js
// 1. 크로미움 브라우저 실행
const browser = await chromium.launch({
    headless: true,
});

// new 탭 열기
const page = await browser.newPage();
```

### 2단계 - 페이지 이동
```js
// 2. 영화 예매 페이지 이동
await page.goto('https://cgv.co.kr/cnm/movieBook/movie', {
      waitUntil: 'networkidle',
      timeout: 30000,
    });
```

### 3단계 - 영화 클릭

자동로그인이 된 상태로 웹에서 테스트 할때랑

크로미움이 띄운 화면이 달랐던 첫번째 고비였습니다.

사실 시크릿탭에서 띄운거랑 거의 동일했고 

추가적으로 영화선택하는 화면, 버튼 기다리기 등등의 처리가 필요했습니다.

```js
// 3. 내가 원하는 영화 버튼 클릭
const movieButton = page
    .locator('button:has(span.title)')
    .filter({ hasText: '영화 제목' });
// 화면에 버튼이 보일때까지 대기
await movieButton.waitFor({
    state: 'visible',
    timeout: 30000
});
// 클릭
await movieButton.click({ force: true });
```

### 4단계 - 로딩 완료 시점까지 기다리기

위에서 분명 영화를 클릭하고 잘 넘어가서

다음 코드들이 동작을 해야하는데

그 사이에 있던 로딩 스피너와 딤드 화면 때문에

로딩이 되는 순간에 나머지 아래 코드들이 다 실행되어버려서 

문제가 없는데 코드가 동작 안하는것처럼 보였던 부분입니다.

waitForSelector를 통해 로딩 스피너가 없어질때까지 

혹은 최대 15초동안 대기하고 타임아웃 시켜버렸습니다.
```js
// 4. 페이지 로딩상태 대기
await page.waitForSelector(
    'span.로딩스피너 선택자',
    { timeout: 15000 }
);
```

### 5단계 - 이제 예매할수 있나?

다른 영화관이나 다른 영화가 그 날짜에 개봉했던건지

이제 곧 열릴꺼여서 그런건지 모르겠지만

페이지에 예약 날짜가 떳다 ! 라는 알림이 와서 들어가봤는데

실제로는 disabled 처리된 날짜만 보이고 클릭은 불가능했습니다.

isVisible, isEnabled 옵션을 제공해줘서

직접 querySelector 안하고 속성 안찾아도 되서 편하더라구요

```js
// 5. 7일버튼 찾기
const dateButton = page
  .locator('button:has(span.number)')
  .filter({ hasText: 7 });
// 버튼이 화면에 노출되었는지 disabled 된건 아닌지 확인
const isVisible = await dateButton.isVisible();
const isEnabled = await dateButton.isEnabled();
```

일단 여기까지는 애초에 거의 다 자바스크립트 로직이기도 했고

플레이라이트를 처음 써봐서 그렇지 대충 있겠다 싶은 기능들은 다 있어서 간단했습니다.

크로니움으로 브라우저를 열고 탭을 켜서 페이지를 이동하고

중간 몇몇 과정들 영화를 클릭하고 로딩을 기다린뒤

예약 페이지에서 내가 원하는 날짜가 있는지 확인하고 종료 !

정도의 로직으로 간단하게 완성했습니다.

## 3. 내가 없더라도 정기적으로 들어가서 확인하기

코드는 내가 생각하는대로 동작하는걸 확인했는데

이걸 이제 알아서 5분마다 한번씩 동작하게 만들려니 잘 모르는 영역이어서 막막헀다.

5분마다 한번씩 도는걸 배치돈다고 하나

이 코드는 노드상에서 돌리면 되겠지

예전에 백엔드 개발자가 크론잡 돌린다고 했던거같은데 이런 역할인건가?

하면서 검색을 했더니 얼추 비슷하게 나왔다.

node를 통해 실행하고 node-cron을 통해서 일정 시간마다 실행할수 있도록 제공했습니다.

```javascript
const cron = require('node-cron');

// 5분마다 실행 (0분, 5분, 10분, 15분...)
cron.schedule('*/5 * * * *', async () => {
  const result = await checkCGV();

  if (result) {
    process.exit(0);
  }
});
```

이런식으로 간단하게 schedule 5분마다 아래 콜백 실행 정도의 코드였습니다.

프로세스를 종료해주는것도 꼭 필요했었던거같고

하나 신기했던게 '*/5 * * * *' 요 문법이 단순하게 5분마다 실행되는

내가 12분에 코드를 돌렸으면 17분 22분 27분 이렇게 돌아가는줄 알았는데

*/5 즉, 분을 5로 나눠서 0으로 떨어지는 정각기준 0분, 5분, 10분 이시점에만 실행이되는 로직이더라구요

## 4. 찾으면 나에게 알려주기

알아서 브라우저 열어서 찾아주는 로직도 짰고

5분마다 실행시켜주는 방법도 찾았으니

이제 내가 항상 컴퓨터 앞에 있는건 아니니까

완료되면 어떻게든 알려주느 방법이 없을까 고민하던차에

카톡은 당연히 지원 안하는거 같고

텔레그램으로 메시지를 보낼수 있게 api를 제공한다는걸 보고

한번 찾아보자 했습니다.

### 1단계 - 텔레그램 봇 생성

뭔가 서비스 개발자도구에서 등록하고 인증받고 발급하고 하는 과정없이

아래처럼 간단하게 봇 생성하고 만들수 있어서 신기했습니다.

1. 텔레그램에서 `@BotFather` 검색
2. `/newbot` 명령어로 봇 생성
3. Bot Token 받기
4. 봇과 대화 시작 (아무 메시지나 전송)
5. `https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates` 접속하여 Chat ID 확인

### 2단계 - 텔레그램 api fetch 만들기

토큰받고, 챗ID 받으면 그걸로 끝

api는 fetch나 axios나 아무거나 편한걸로 주어진 토큰들 넣고 호출했습니다.

```javascript
await fetch(`https://api.telegram.org/bot${TELEGRAM_BOT_TOKEN}/sendMessage`, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
        chat_id: TELEGRAM_CHAT_ID,
        text: message,
        parse_mode: 'HTML'
    })
});
```
